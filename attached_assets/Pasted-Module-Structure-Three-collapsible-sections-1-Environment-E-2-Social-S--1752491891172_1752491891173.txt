Module Structure:

- **Three collapsible sections**:
  1. **Environment (E)**
  2. **Social (S)**
  3. **Governance (G)**

- Each section expands to reveal **ESRS-aligned topics**. When a topic is selected, a structured input form appears for entering relevant KPIs and metadata.

Each ESG section must contain a list of disclosure topics (as per ESRS) and their respective KPIs. Use the following structure for each KPI:

#### Example for a KPI like "Scope 1 GHG Emissions":

- **KPI Name**: Auto-filled dropdown
- **Metric Type**: Dropdown ‚Üí [Quantitative, Qualitative, Monetary, Ratio]
- **Unit of Measure**: Dropdown ‚Üí [tCO‚ÇÇe, kWh, %, EUR, Count, etc.]
- **Reference Standard**: Multi-select ‚Üí [GHG Protocol, ISO 14064, EU Taxonomy, Custom]
- **Baseline Year**: Date picker

#### Additional Metadata Fields:

- **Data Owner / Provider**: Dropdown ‚Üí [Sustainability Officer, HR, Finance, Legal, External Consultant]
- **Source Type**: Multi-select ‚Üí [ERP, Utility Bill, Sensor, Supplier Report, HRIS, Manual]
- **Collection Frequency**: Dropdown ‚Üí [Real-time, Monthly, Quarterly, Annually]
- **Collection Method**: Dropdown ‚Üí [Manual Entry, Excel Upload, API Integration, Survey Tool]
- **Assurance Level**: Dropdown ‚Üí [None, Limited, Reasonable, Third-Party Certified]
- **Verification Status**: Dropdown ‚Üí [Not Verified, Internally Reviewed, Third-Party Verified]
- **Confidentiality Level**: Dropdown ‚Üí [Public, Internal Only, Management Restricted]
- **Supporting File Upload**: File input for CSV, XLSX, or PDF

### üß≠ Navigation & UI Behavior:

- When the user clicks **Environment / Social / Governance**, the corresponding disclosure topics are shown.
- Selecting a topic (e.g., E1 ‚Äì Climate Change) shows a list of pre-defined KPIs.
- For each KPI, an input form should appear with all fields mentioned above.
- Users can:
  - **Add New KPI**
  - **Edit Existing KPI**
  - **Delete KPI**
  - **Upload supporting documentation**
